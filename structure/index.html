Java Servelets

  • Java objects that are intended to pay the role of server components in client server communications.
  • Handles client requests by generatig responses to such requests
  • Java Servelet is also a part of java EE specification.
  • Java Servelet is hosted in a servelet container that provides the environment in which the servelet runs, as well as controls the servelet lifecycle

    Initialization

    • invoked only once and is intended for any startup initialization code for the servlet
    • init() method

    Request handling

    • service() method
    • invoked for each client request
    • servlet performs whatever logic necessary in order to serve the request and generate a response.
    • multiple client requests are typically handled by a single, multi-threaded servlet instance;; thus, care must be exercised to ensure that the code being run by the servlet is thread-safe.

    Destruction

    • destroy() method
    • invoked before the servlet instance is unloaded and is untended for any housekeeping code that the servlet may need to run.

HTTPSERVLET

  • Java servlet used to handle HTTP requests and generateHTTP resposes.
  • Hosted in a web container which is a component in a java application server.
  • The service() method call is routed to doXXX() call, depending on the HTTP request method, e.g., doGet(), doPost(), doHead(), doPut(), doDelete(), etc.
  • HttpServletRequest - Object representation of the HTTP request sent by the client and received by the servlet. It is used to access information from the request message.
  • HttpServletResponse - Object representation of the HTTP response generated by the servlet and sent back to the requesting client. It is used to set information in the response message.
